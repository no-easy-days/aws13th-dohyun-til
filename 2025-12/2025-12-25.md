## BRANCH
- 개발에 각각 다른파트 작업을 진행할때, 분기시켜 새 루트를 만들고 그 branch에서 작업을 진행한다.
### BRANCH STRATEGY
- 이때, 각 branch별로 github flow, gitflow 등과 같은 브랜치 전략을 사용하여,
  각 branch에 의미 있는 이름 ex)feature/login 과 같이 알아보기 쉽게 할 수 있다.
--------------
### HEAD
- 내가 어떤 branch인지, 혹은 commit에서 작업을 진행하고 있는지 터미널상에서 쉽게 알아볼 수 있다.
### chekout/switch
- checkout 혹은 switch로 원하는 branch로 옮겨다니면서 진행할 수 있는데, switch -c 와 같이 새 branch를 생성하면서 이동하여 빠르게 작업을 진행할 수 있다.

## GITHUB
- 이러한 GIT을 직접 관리할 수 있게 함
-----------
### git remote
- 로컬 환경을 원격으로 온라인으로 업로드
### git fetch
### git push
- 로컬에서의 변경사항을 온라인으로 업로드
  - 이때, pull로 변경사항을 먼저 인지해야 push가 가능함.
### git commit
- add후에 commit을 날려줌
### git pull
- 변경사항이 존재한다면, 먼저 가져오도록 한다.

### git merge
- pull로 당겨오고, push전에 변경사항이 만약 같은 파일에 존재한다면,
- merge로 수동 혹은 자동 병합을 진행해준다.
